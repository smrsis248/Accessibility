Features For Buttons

Click to Reveal

This content is revealed with the button click. It is helpful to put very descriptive text for buttons. "Click to Reveal" is more descriptive than just "Click Me" or "Button", etc. Also, it is helpful to have this content under the button because screen readers don't automatically go back to the html preceding something they have already read. If this hidden content precedes the button, screen readers wouldn't go back to read it and users would think the button did nothing.

It is acceptable to create buttons using the anchor tag. The code for the "Click to Hide" button is:

< href="#" class="button" id="button2">Click to Hide</a>

Images

It is important to use the "alt" tag when including images on your webpages. "Alt" stands for Alternative Text. The alt text for this image could read something like this: "Snow-dusted landscape with a pristine lake in front of a mountain range".

More helful hints on how to write alt text can be found in the article: The Definitive Guide To The Alt-Text Field.

CheckBoxes

For this example, there are two important components to the input tag (what is used to make the checkbox): "type" and "required". "Type" tells the screen reader what kind of element it is looking at. "Required" is either true or false and tells the user that, in this case of the checkbox, either it is required to be clicked or not. Also, when coding the input and label tags, make sure the "id" for the input tag matches the "for" of the label tag.


The screen reader would read the above as something like:

Option 1 Required checkbox not checked
Option 2 checkbox not checked

Links

The text of a link should say exactly what the link is going to. For example, this is a terrible way to link to the WCAG page:

Click here to go to the WCAG webpage.

A better way to go about it would be to write something more like:

More information about WCAG could be found on the WCAG webpage.

Also, it isn't a bad practice to use the "target" attribute in the anchor tag, which will open the link in a new tab:

<a href="#" target="_blank">Link Text</a>

Input Box

Below are two ways to code input boxes for forms. One way is called 'explicit' and the other is 'implicit'. For 'explicit', there is a text label with a "for" attribute (like used in the checkbox example). For 'implicit', there is no "for" attribute and both the label text and the form input are within the label element. Both options are acceptable for accessibility purposes.

The above is an example of the way to code an 'explicit' label. The code looks like:

<label for="name">Name</label>
<input type="text" id="name">

The above is an example of the way to code an 'implicit' label. The code looks like:

<label>Name <input type="text" role="combobox"></label>
